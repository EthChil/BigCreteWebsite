Big Crete Raytracer

Overview

As one of my first FPGA projects this one has been in development for a while largely due to me constantly going back and re-architecting it. The goal of this project is to make a digital design that can raytrace a basic scene completely in hardware with no softcore. This means scene geometry is stored in bram and is directly used as an input to generate the scene. For now the goal is a sphere with a single light source and a ground plane.

Target Specs

  • 720p rendering
  • Double buffered frames in ram
  • Display over VGA for it's simplicity
  • 60fps video output regardless of raytracing compute time
  • Physically accurate raytracing

Custom DAC Board - April 2021

I designed and ordered a custom DAC PCB with a DAC chip capable of driving the display over VGA at full 24bit colour.

Double Buffered Frames in RAM - March 2021

This was a surprisingly challenging milestone as the RAM isn't fast enough to feed the display so a number of buffering stages and state machines were added to coordinate between when data needs to be pulled for updating the display and writing newly generated pixels to memory.

Display Output - March 2021

First step was to build an RTL module capable of driving a display over VGA. Since VGA is analog I cut up a VGA cable and built a rudimentary DAC using resistors on a breadboard to get a simple 6bit colour output for testing.
